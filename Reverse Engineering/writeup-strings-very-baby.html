02

Analisis File

File yang diberikan bernama chall tanpa ekstensi. Langkah pertama adalah mengidentifikasi jenis file tersebut.

file identification

[METADATA]

Dari analisis header, diketahui bahwa file chall merupakan sebuah ELF 64-bit LSB executable untuk arsitektur x86-64 di Linux. Yang paling penting: file ini berstatus "not stripped", artinya debug symbol masih tersedia dan string di dalam binary tidak dihapus.

Entropy rendah (1.6039) juga menjadi indikator kuat bahwa binary ini tidak di-obfuscate maupun di-pack — kontennya mudah dibaca secara langsung.

05

Penjelasan Teknis

Secara teknis, soal ini menguji pemahaman dasar tentang format binary ELF. Flag disimpan sebagai plaintext string di dalam binary — sebuah teknik yang paling umum dijumpai pada soal Reverse Engineering tingkat pemula.

1

Binary Tidak Di-strip

File berstatus not stripped, artinya semua string literal — termasuk flag — masih dapat dibaca langsung dari segmen .rodata atau .data pada binary.

2

Entropy Sangat Rendah

Nilai entropy 1.6039 jauh di bawah threshold packing/obfuscation (~7.0+). Ini mengkonfirmasi binary tidak diproteksi dengan UPX, XOR, atau teknik encoding apapun.

3

Strings Analysis

Teknik strings bekerja dengan mengekstrak semua sequence karakter ASCII yang dapat dibaca dari binary. Flag format LKS{...} terdeteksi secara langsung.

4

Auto-Fix ELF Header

Tool sfores juga melakukan auto-fix pada header ELF (output: fixed_chall.elf), namun tidak diperlukan untuk mendapatkan flag karena strings sudah berhasil diekstrak.

06

Flag

🚩 captured flag
ADB{ah_yes_classic_strings!}
Lesson learned: Selalu cek strings pada binary ELF terlebih dahulu sebelum melakukan analisis yang lebih kompleks. Soal "very-baby" seperti ini sering kali menyembunyikan flag langsung di dalam binary tanpa enkripsi atau obfuscation apapun.
